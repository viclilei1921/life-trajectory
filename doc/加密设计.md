# 生命轨迹 —— 加密设计

## 一、与整体架构的对齐

### 1.1 设计原则

| 原则 | 说明 | 来源 |
| --- | --- | --- |
| **Local-First** | 密钥与密文都在本地；无云端找回密码 | 整体设计 §一 |
| **降低记录摩擦** | 解锁一次即可连续写入；不在每条记录做 Argon2 | 整体设计 §三 |
| **诚实安全分级** | 无密码 = 不安全；给用户明确选项与文案 | 本文 §三 |
| **Rust 持有密钥** | DEK 只在后端内存；前端只知道 `isUnlocked` | Tauri 分层 |

### 1.2 在分层架构中的位置

```
┌─────────────────────────────────────────┐
│  Svelte UI                               │
│  - 解锁页 / 设置页                        │
│  - vaultStore: { isUnlocked, mode }      │
└───────────────┬─────────────────────────┘
                │ Tauri command
┌───────────────▼─────────────────────────┐
│  server/handlers + services              │
│  VaultService / EntryService             │
│  └─ 读写前检查 VaultSession              │
├──────────────────────────────────────────┤
│  crypto/（纯原语，无业务）                │
│  blob.rs / core.rs / error.rs            │
├──────────────────────────────────────────┤
│  Repository（sqlx）                      │
│  vault_config 表 + entries 密文字段     │
└──────────────────────────────────────────┘
```

Vault 会话、DEK 包装编排位于 [`server/services/vault.rs`](../src-tauri/src/server/services/vault.rs)；`crypto/` **不**承载 vault 业务逻辑。

## 二、核心架构：DEK + KEK + 会话

```
用户密码 ──Argon2id──▶ KEK（32 字节，密钥加密密钥）
                           │
                           ▼ 加密/解密
                      Wrapped DEK（持久化）
                           │
                           ▼ 解锁后放内存
                        DEK（32 字节随机，数据加密密钥）
                           │
                           ▼ 加解密每条 content
                   entries.content_encrypted (BLOB)
```

### 2.1 三层职责

| 层级 | 作用 | 生命周期 |
| --- | --- | --- |
| **用户密码** | 仅用于解包 / 重包装 DEK | 启动或改密码时输入 |
| **DEK** | 加密日记正文 | 解锁后常驻 Rust 内存，锁定时 zeroize |
| **每条 nonce** | 同明文不同密文 | 存在每条 BLOB 头部 |

### 2.2 为何分离 DEK / KEK

- 用户启动时输**一次**密码即可连续记录。
- **改密码只重包装 DEK**，不必重加密全部 entries。
- 每条写入**不做 Argon2**，显著降低写入延迟。

### 2.3 算法选型

| 用途 | 算法 | 参数 |
| --- | --- | --- |
| 密码 → KEK | Argon2id | m=15_000 KiB, t=3, p=1, out=32（与现有 `core.rs` 一致） |
| DEK 包装 / 正文 | ChaCha20 流加密 | 256-bit key，12-byte nonce；DEK 包装明文附 `VERIFY_MAGIC` 拒绝错密码 |
| 随机数 | `rand` crate | OS CSPRNG |
| 内存清零 | `zeroize` crate | 锁定 / Drop 时清除 DEK、KEK |

## 三、两种安全模式

不要假装「没密码也安全」。UI 与 `vault_config.mode` 保持一致。

### 3.1 模式 A：`none`（无保护，**默认**）

```
content → 明文 TEXT 存入 entries.content
```

- 标注：「未加密，仅本地存储」
- 适合：先用起来；后续可在设置页升级为 `password`
- `wrapped_dek` 为空；`is_encrypted = 0`
- `VaultStatus.isUnlocked` 恒为 `true`

### 3.2 模式 B：`password`（主密码）

```
用户密码 → Argon2 → KEK → 包装 DEK → vault_config.wrapped_dek
```

- 启动时 session **locked**；用户 unlock 后 DEK 进内存
- 拷走数据库无密码难以暴力破解（Argon2 拖慢）
- **必须告知**：忘记密码 = 数据无法恢复（Local-First 无云端找回）

### 3.3 首次启动流程

```
首次启动（migration 默认 mode = none）
  │
  ├─ 直接使用 → 明文存储，无解锁页
  │
  └─ 设置页「启用主密码」
        → vault_setup_password
        → 生成 DEK + salt + wrapped_dek 写库
        → mode = password，session 已解锁
```

### 3.4 模式对比（设置页展示用）

| | none | password |
| --- | --- | --- |
| 记密码 | 否 | 是 |
| 防拷走 db | 否 | 是 |
| 防本机恶意程序 | 否 | 否（解锁后同样暴露） |
| 忘记密码 | — | 数据不可恢复 |

## 四、数据模型

### 4.1 `vault_config`（单行配置）

见 [`migrations/0001_initial.sql`](../src-tauri/migrations/0001_initial.sql)（entries + vault_config 合并为单一初始 migration）：

```sql
CREATE TABLE vault_config (
  id              INTEGER PRIMARY KEY CHECK (id = 1),
  mode            TEXT NOT NULL CHECK (mode IN ('none', 'password')),
  kdf_salt        BLOB,           -- password 模式：Argon2 盐（16 字节）
  wrapped_dek     BLOB,           -- password 模式：KEK 包装后的 DEK（v2 BLOB）
  wrap_nonce      BLOB,           -- 预留；当前 v2 BLOB 内嵌 nonce
  crypto_version  INTEGER NOT NULL DEFAULT 1,
  created_at      INTEGER NOT NULL,
  updated_at      INTEGER NOT NULL
);

INSERT INTO vault_config (id, mode, crypto_version, created_at, updated_at)
VALUES (1, 'none', 1, unixepoch(), unixepoch());
```

**`wrapped_dek` 格式**（password 模式，与 `encrypt_v2` 一致）：

```
MAGIC_TITLE(8) + VERSION(1) + NONCE(12) + ciphertext(40)
```

- 包装明文 = `DEK(32) ‖ VERIFY_MAGIC`（`VERIFY_MAGIC = b"LTVAULT1"`，可公开）
- `ciphertext = ChaCha20(KEK, nonce, DEK ‖ VERIFY_MAGIC)`，KEK = Argon2id(password, kdf_salt)
- 解锁时校验明文尾部 magic；对不上 → `VAULT_INVALID_PASSWORD`（流密码本身不拒绝错密钥，靠校验串门禁）
- 旧版仅包装 32 字节 DEK、无 magic 的 `wrapped_dek` 无法解锁（MVP：重设密码 / 清库）

### 4.2 变更：`entries` 表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `content` | TEXT NOT NULL DEFAULT '' | **明文模式**下的正文；加密模式下恒为空字符串 |
| `content_encrypted` | BLOB NULL | 加密正文；`password` 模式且有 DEK 时有值 |
| `is_encrypted` | INTEGER NOT NULL DEFAULT 0 | 0=明文，1=密文（便于迁移与查询） |
| `title` | TEXT NULL | **第一版明文**（时间轴列表需要） |
| 其余字段 | — | 与整体设计一致，**保持明文**（时间、地点、标签、心情等） |

**字段加密策略（第一版）**

| 字段 | 是否加密 | 原因 |
| --- | --- | --- |
| `content` | 是（核心） | 日记正文 |
| `title` | 否 | 时间轴、FTS 需要；敏感用户阶段 2 可选加密 |
| `happened_at`、标签、心情等 | 否 | 排序、筛选、状态曲线 |
| `people.name`、`locations.name` | 否 | 关系/地图视图；后续按需 |

### 4.3 密文 BLOB 格式（单条 entry）

**新格式**（DEK 模型，无 per-entry salt）：

```
偏移   长度   内容
0      8      MAGIC = b"chacha20"
8      1      VERSION = 0x02        -- 0x01 为旧演示格式（含 salt）
9      12     NONCE
21     N      CIPHERTEXT（与明文等长）
```

- 加密前：UTF-8 编码的 Markdown 字符串
- 同一 DEK 下**每条必须随机 nonce**；nonce 复用会灾难性失败
- `VERSION = 0x01`（`MAGIC + VERSION + SALT + NONCE + cipher`）保留给 `simple.rs` 演示/导出，**不写入 entries 表**

### 4.4 数据目录（与整体设计一致）

```
生命轨迹数据/
├── life.db              # 含 vault_config + entries 密文
├── media/               # 第一版明文；后续可选加密
└── thumbnails/
```

### 5.2 核心类型（[`services/vault.rs`](../src-tauri/src/server/services/vault.rs)）

```rust
// models/vault.rs
pub enum VaultMode { None, Password }

// services/vault.rs
pub struct VaultSession {
    dek: Option<Zeroizing<[u8; 32]>>,
    crypto_version: i32,
}

impl VaultSession {
    pub fn is_unlocked(&self) -> bool;
    pub fn require_dek(&self) -> Result<&[u8; 32], AppError>;  // → VAULT_LOCKED
    pub fn lock(&mut self);
    pub fn unlock_with_password(&mut self, password, salt, wrapped_dek) -> Result<()>;
    pub fn create_password_wrapped(password) -> Result<(Self, salt, wrapped_dek)>;
    pub fn rewrap_with_new_password(...) -> Result<(new_salt, new_wrapped)>;
}

pub struct VaultService;  // bootstrap / unlock / lock / setup_password / change_password / …
```

## 六、会话管理

### 6.1 状态机

```
App 启动（VaultService::bootstrap_session）
  │
  ├─ mode = none      → 空 session；is_unlocked 对外恒 true
  └─ mode = password  → 空 session（locked）→ 解锁页
        │
        └─ unlock(password)
              ├─ Argon2 → KEK
              ├─ unwrap wrapped_dek → DEK
              └─ VaultSession.dek = Some(...)

读写 entries（EntryService）
  │
  ├─ none:     明文读写 entries.content
  ├─ password: require_dek → encrypt_v2 / decrypt_v2 → content_encrypted
  └─ INSERT / SELECT 对前端仍返回明文 content

锁定（DEK zeroize）
  ├─ 用户点击「锁定」（vault_lock）
  ├─ 空闲 N 分钟（阶段 3，可配置）
  └─ 应用退出
```

### 6.2 Tauri 集成

```rust
// server/mod.rs bootstrap
let session = VaultService::bootstrap_session(&pool).await?;
handle.manage(AppState {
    db: pool,
    vault: Mutex::new(session),  // tokio::sync::Mutex
});
```

- `none` / `password` 启动时均创建 **locked 空 session**（`none` 模式对外 status 仍报 unlocked）
- password 模式需前端调用 `vault_unlock` 后才可读写加密 entry

### 6.3 关键原则

- DEK **绝不**通过 IPC 传给前端
- 前端仅同步：`{ mode, isUnlocked, cryptoVersion }`
- 锁定后：列表仍可显示标题/时间；点进详情提示「请先解锁」
- `password` 模式：未解锁时 `create_entry` 返回明确错误码 `VAULT_LOCKED`

## 七、关键业务流程

### 7.1 解锁（password 模式）

```
输入 password
  → 读 vault_config.kdf_salt
  → KEK = Argon2id(password, salt)
  → plain = decrypt_v2(wrapped_dek, KEK)
  → 校验 plain 尾部 == VERIFY_MAGIC；失败则 VAULT_INVALID_PASSWORD
  → DEK = plain[0..32]
  → 写入 VaultSession
  → 可选：重建 FTS 索引（见 §九）
```

验证失败：统一提示「密码错误」。正文条目仍为无认证的 ChaCha20；校验串只保证解锁门禁，不代替 AEAD。

### 7.2 新建记录（加密模式）

```
create_entry({ title, content, happened_at, ... })
  → session.require_dek()
  → blob = entry::encrypt(content.as_bytes(), dek)
  → INSERT entries (content='', content_encrypted=blob, is_encrypted=1, ...)
  → FTS：写入明文到 entries_fts（仅内存索引，见 §九）
```

### 7.3 改密码（password 模式，已解锁）

```
1. 验证 old_password → 解包 DEK（必须与内存 DEK 一致）
2. new_KEK = Argon2id(new_password, new_salt)
3. 重新 wrap DEK → 更新 wrapped_dek, kdf_salt, wrap_nonce
4. entries 无需变动
```

### 7.4 模式迁移

| 迁移 | 步骤 |
| --- | --- |
| **none → password** | `vault_setup_password` → 生成 DEK → 批量加密 entries → 写 wrapped_dek |
| **password → none** | 需用户确认风险 → 批量解密 → 清空 wrapped_dek → mode = none |
| **password 改密码** | 已解锁 → `rewrap_with_new_password` → 只更新 salt/wrapped_dek |

批量迁移在 **tokio 后台任务**执行，前端显示进度；迁移中断可恢复（按 `is_encrypted` 逐条处理）。

## 九、错误码

| 码 | 含义 |
| --- | --- |
| `VAULT_LOCKED` | 需解锁 |
| `VAULT_ALREADY_UNLOCKED` | 重复解锁 |
| `VAULT_INVALID_PASSWORD` | 密码错误 |
| `VAULT_MODE_MISMATCH` | 操作与当前 mode 不符 |
| `CRYPTO_INVALID_BLOB` | 密文损坏或版本不支持 |
| `MIGRATION_IN_PROGRESS` | 批量迁移中，请稍后 |
